# 🛠️ MLIT Photo Pro: プロジェクト固有ルール

本ファイルは「MLIT Photo Pro」プロジェクトにおける技術スタックおよびコーディング規約を定義します。AIの全般的な基本動作やガイドラインについてはグローバル設定の `~/.gemini/GEMINI.md` を参照してください。

## 1. フロントエンド / UI アーキテクチャ

1. **基本構成 (Vite + React)**
   - 高度な状態管理とコンポーネント指向の開発を推進するため、フロントエンドの基本構成は **Vite + React** を使用します。
   - レンダリング最適化やファイル構造の明確化を意識し、保守性の高い設計を維持します。
2. **スタイリング (TailwindCSSの活用)**
   - スタイリングには **TailwindCSS** を全面的に活用します。
   - ユーティリティクラスを基本としつつ、デザインの再利用性やDRY原則に従い `index.css` なのグローバルCSSも必要に応じて併用します。
3. **モダンで洗練されたUI/UXデザイン**
   - 単なる機能実装にとどまらず、**Vibe Coding** の精神に則り、ユーザーの目を惹く美しいデザイン（調和の取れた配色、ダークモード対応、グラスモーフィズム、適切なマイクロアニメーションなど）を第一稿から実装に組み込みます。

## 2. コード品質と命名規則

1. **肯定的なアプローチ**
   - AIへのプロンプトやコーディングルールを追記する際は、「〜しないで」という否定系よりも「〜すること」という肯定系での直接的な記述を心がけます。
2. **命名規則 (キャメルケース)**
   - 変数名や関数名は直感的・意味論的な英単語を使用し、原則としてキャメルケース (`camelCase`) で統一します。
3. **コメント (日本語)**
   - 複雑なロジックや、後日参照する際に意図が伝わりにくい処理には、必ず**日本語**で明確なコメントを付与します。
