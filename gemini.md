# Gemini (Antigravity) 開発ガイドライン

このファイルは、AIエージェント（Antigravity）がこのプロジェクトで自律的かつ効率的に開発を進めるための「ベストプラクティス設定ファイル」です。

## 🤖 基本スタンス (AI Behavior & Vibe Coding)
- **ハイレベルな目標駆動**: ユーザーは「何を実現したいか（What）」を伝え、AIが「どう実装するか（How）」を計画・実行します。（いわゆる "Vibe Coding" の実践）
- **マルチサーフェス実行**: AIはコードの編集だけでなく、必要に応じて自律的にターミナルでのコマンド実行（npm install等）や、ブラウザを立ち上げてのUI動作確認を行います。
- **プロアクティブな提案と安全第一**: エッジケースの考慮やUI/UXの改善案を自発的に実装しつつ、破壊的な変更や重要なアーキテクチャの修正前には必ず人間のレビューを求めます。
- **既存コードの保護（最小限の変更）**: 既存のコードに手を入れる際は、**変更箇所を必要最小限（最小スコープ）に留める**こと。依頼されていない不要なリファクタリングや、ユーザーが意図しない設計・機能の変更は絶対に行わないこと。

## 💻 技術スタックとコーディング規約 (Tech Stack & Coding Standards)
- **フロントエンド / UI**:
  - 用途に合わせてシンプルで高速な構成（Vanilla CSS + JS）、またはモダンなフレームワーク（Next.js/Vite）を適宜提案・使用します。
  - TailwindCSS はユーザーの明示的な指示がある場合のみ使用します。
  - **デザイン**: ユーザーの目を引くモダンで洗練されたUI（美しい配色、グラスモーフィズム、滑らかなマイクロアニメーション）を初期実装から提供します。
- **コード品質**:
  - ガイドラインやルールの記述は、「〜しないで」という否定形よりも「〜すること」という肯定形で直接的な指示を心がけます。
  - 変数名や関数名は直感的な英単語（キャメルケース）を使用し、複雑な処理には意図が伝わる日本語コメントを付与します。

## 📝 自律型ワークフロー (Autonomous Workflow)
1. **計画 (Planning)**: 中〜大規模のタスクではいきなりコードを書かず、まず `implementation_plan.md` 等で設計を可視化し、合意を形成します。
2. **実行 (Execution)**: 指示に基づき、自律的に関連ファイルの編集とコマンドの実行を並行して進めます。
3. **検証 (Verification)**: ブラウザ画面やターミナルの出力をAI自身が確認し、自己デバッグを行った上で最終的な結果（テスト結果やUIのスクショ）を報告します。
